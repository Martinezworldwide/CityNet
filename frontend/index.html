CityNet Traffic Operations Dashboard

Instructor guide (Java backend verification)
This page calls a live Java/Spring Boot backend on Render at the URL above. Use the panels below to exercise: heap-prioritized alerts (Java PriorityQueue), a recursive Dijkstra emergency router, and a recursive predictive traffic simulator. Open Developer Tools → Network to see https://citynet-backend.onrender.com/... requests.

Heap-Prioritized Alerts

How to test (alerts)
1. Type any incident description and a severity (integer; higher = higher priority).
2. Click Create Alert to send POST /alerts to the Java backend (stored in a max-heap PriorityQueue<Alert>).
3. Click Get Next Alert to call GET /alerts/next and pop the highest-priority alert from the heap.

Emergency Route (Recursive Dijkstra)

How to test (routes)
1. Provide real intersection identifiers in Start and Target (matching your road-network data).
2. Click Request Emergency Route to call GET /routes/emergency, which runs the recursive RecursiveDijkstra algorithm in Java using a min-heap PriorityQueue, blocked-road flags, and congestion penalties supplied by your data source.
3. To drive it directly with your own graph, use POST /routes/emergency (for example via Postman) with a JSON body shaped like:
{
  "startIntersectionId": "START_INTERSECTION_ID",
  "targetIntersectionId": "TARGET_INTERSECTION_ID",
  "edges": [
    {
      "id": "EDGE_ID_1",
      "fromIntersectionId": "START_INTERSECTION_ID",
      "toIntersectionId": "INTERSECTION_B",
      "baseTravelTime": BASE_TRAVEL_TIME_SECONDS,
      "blocked": false
    }
  ],
  "congestionPenalties": {
    "EDGE_ID_1": CONGESTION_PENALTY_SECONDS
  }
}
Replace the identifiers and numeric values with real data from your course dataset or lab.

Predictive Traffic Simulation

How to test (simulation)
1. Paste a JSON payload describing your real initial traffic state and parameters, then click Run Simulation to call POST /simulate. The Java backend runs the recursive PredictiveTrafficSimulator over a time horizon.
2. Example JSON shape (fill in real intersection IDs and congestion values):
{
  "initialState": {
    "currentTime": 0,
    "congestionByIntersection": {
      "INTERSECTION_A": CONGESTION_LEVEL_A,
      "INTERSECTION_B": CONGESTION_LEVEL_B
    },
    "signalByIntersection": {
      "INTERSECTION_A": "GREEN",
      "INTERSECTION_B": "BALANCED"
    },
    "history": []
  },
  "horizon": 30,
  "timeStep": 5,
  "congestionThreshold": CONGESTION_THRESHOLD_VALUE
}
3. The response shows a timeline of TrafficState objects and signalRecommendations derived from the recursive prediction.
